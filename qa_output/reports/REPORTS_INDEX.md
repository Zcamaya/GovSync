# QA Reports Index

This index lists the core QA reports generated by the QA engine and what to look for in each.

- `crawler_snapshot.json` — full discovered widget tree snapshot.
- `navigation_map.json` — discovered navigation targets and sidebar state.
- `page_capture.json` — screenshots and metadata for each captured page/dialog. (See `dialog_employeedetailsdialog` capture.)
- `page_capture_debug.json` — debug-mode captures.
- `VISUAL_REPORT.html` — combined visual summary you can open in a browser.
- `layout_issues.json` / `LAYOUT_REPORT.md` — widgets detected outside their expected parents (`out_of_parent`).
- `FIX_PRIORITY.md` — prioritized list of layout/accessibility items.
- `accessibility.json` — labels, buttons, and fields the inspector read; check for missing `object_name` or `text`.
- `text_visibility.json` / `TEXT_VISIBILITY_REPORT.md` — widgets with missing/empty text.
- `performance.json` / `PERFORMANCE_REPORT.md` — basic performance checks.
- `tables.json`, `forms.json` — table and form specific inspection results.
- `widget_tree.json` / `WIDGET_TREE.md` — full widget tree in JSON and a readable markdown.

Quick actions

- To open the visual report in your default browser:

```powershell
Start-Process "$(Resolve-Path qa_output/reports/VISUAL_REPORT.html)"
```

- To find source for a widget named in `layout_issues.json` or `page_capture.json`, search the `widgets/` folder for the class or object name.
